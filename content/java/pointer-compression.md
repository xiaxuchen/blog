# 压缩指针
由于64位机器会使得jvm采用64位存储对象引用，相对于32位机器有着相当大的内存浪费，而在64位机器上，jvm的对象会进行对齐填充到8字节，因此对象的地址都是8的倍数，因此压缩指针采用32位存储对象的引用，而其单位是8字节，因此可以表示
2 ^ 35 = 32 G 的内存。也因此，我们不能让jvm的寻址地址超过32G，即`Xmx<=32G`.
